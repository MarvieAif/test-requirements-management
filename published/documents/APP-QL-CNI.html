2.0 UI Behavior

2.1 UI Behavior APP-QL-CNI-011

If exists, Data can be fetched from DB for applicant and co-applicant (if applicable) and loaded into fields otherwise defaults (if applicable) are loaded into fields.

2.2 UI Behavior APP-QL-CNI-012

Data can be saved into DB for applicant and co-applicant (if applicable).

2.3 UI Behavior APP-QL-CNI-013

The tab names are "Main Applicant" and "Co-applicant" (if applicable). Main Applicant and Co Applicant displays the sections Contact, Current Address & Previous Address (if applicable).

2.4 UI Behavior APP-QL-CNI-014

The tab names are "Main Applicant" and "Co-applicant" (if applicable). Co-Applicant displays the sections Contact, Current Address & Previous Address (if applicable). However only co-applicant has the section "Does the co-applicant share the same address with the applicant?"

2.5 UI Behavior APP-QL-CNI-015

If co-applicant chooses "yes" for "same address", then main applicant changes their address, the previous radio choice should not persist and switch to "no" automatically. If co-applicant and main applicant both input the same address (But not living here field), the radio should automatically switch to "yes". If co-applicant chooses "yes" for "same address", under the section "Does the co-applicant share the same address with the applicant?" the current address section should be disabled, and the address saved into DB should be the same as the main applicant

2.6 UI Behavior APP-QL-CNI-016

If applicant/co-applicant search for the address and click on an address from the drop-down, the "street number", "city", "postal code", "street name", "province" and possibly "unit" fields should be automatically filled.

2.7 UI Behavior APP-QL-CNI-017

If applicant/co-applicant selects the "province" to be other than "Alberta", "British Colombia", "Ontario" or "Quebec", a message will be toasted, preventing them from proceeding.

2.8 UI Behavior APP-QL-CNI-018

If main applicant passes validation but co-applicant doesnt, tab should automatically switch to co-applicants and vice versa. Applicable inline errors show and applicable toasts show.

2.9 UI Behavior APP-QL-CNI-019

If main applicant passes validation and (if applicable) co-applicant passes, then relevant data saves, toasts and inline errors are cleared and user is able to move on to the next section.
